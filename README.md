# Вариант 2 – Broken Shop

## Постановка

Нужно проанализировать и исправить небольшой интернет-магазин на `React + MUI + RTK` с архитектурой `FSD`.

## Страницы

- `/` — главная
- `/product/:id` — страница товара
- `/cart` — корзина
- `/favorites` — избранное
- `/404` — страница 404

## Запуск

```bash
npm install
npm run dev
```

## Что проверить и исправить

### 1. Часть ссылок в приложении ведёт на некорректные маршруты

#### Шаги воспроизведения

Кейс 1:
1. Открыть приложение.
2. В шапке нажать `Каталог`.

Кейс 2:
1. Открыть приложение.
2. В адресной строке вручную открыть неподдерживаемый маршрут, например `/random-page`.

Кейс 3:
1. Открыть главную страницу.
2. Нажать на верхнюю цветную область любой карточки товара.

#### Фактический результат

- ссылка `Каталог` ведёт на некорректную страницу;
- неподдерживаемый маршрут открывает не страницу 404, а другой экран;
- клик по верхней части карточки ведёт на маршрут, который приложение не обслуживает корректно.

#### Ожидаемый результат

- каждая ссылка в интерфейсе должна вести на валидную страницу;
- для неподдерживаемого маршрута должна открываться страница 404;
- клик по карточке товара должен открывать страницу выбранного товара.

---

### 2. При переходах по страницам очищаются корзина и избранное

#### Шаги воспроизведения

Кейс 1:
1. Открыть главную страницу.
2. Добавить любой товар в корзину.
3. Убедиться, что в шапке обновился счетчик корзины.
4. Перейти на другую страницу, например в корзину или обратно на главную.

Кейс 2:
1. Открыть главную страницу.
2. Добавить любой товар в избранное.
3. Убедиться, что в шапке обновился счетчик избранного.
4. Перейти на другую страницу, например в корзину, в избранное или обратно на главную.

#### Фактический результат

После перехода на другую страницу добавленные товары исчезают из корзины и из избранного.

#### Ожидаемый результат

Навигация внутри приложения не должна сбрасывать корзину и избранное.

---

### 3. Блок `Смотреть рядом` меняет адрес, но не обновляет страницу товара

#### Шаги воспроизведения

1. Открыть страницу любого товара, например `/product/101`.
2. В блоке `Смотреть рядом` выбрать другой товар.

#### Фактический результат

Адрес в браузере меняется, но содержимое страницы остаётся прежним или обновляется некорректно.

#### Ожидаемый результат

После выбора соседнего товара должна открываться его карточка с актуальными данными.

---

### 4. Для несуществующего товара нет понятного сценария

#### Шаги воспроизведения

1. Открыть маршрут `/product/999`.

#### Фактический результат

Пользователь не получает понятного сообщения о том, что товар не найден.

#### Ожидаемый результат

Если товара не существует, приложение должно показать понятное fallback-состояние: сообщение `Товар не найден` или страницу 404.

---

### 5. После обновления страницы избранное очищается

#### Шаги воспроизведения

1. Открыть главную страницу.
2. Добавить любой товар в избранное.
3. Убедиться, что в шапке обновился счетчик избранного.
4. Обновить страницу.

#### Фактический результат

После обновления страницы список избранного очищается.

#### Ожидаемый результат

Обычное обновление страницы не должно приводить к потере избранных товаров.
